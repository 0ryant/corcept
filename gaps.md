## Goal

Productise `corcept` as a hook-time governance surface with a callable MCP
interface and durable receipts.

## Missing

- Canonical `corcept mcp serve` or mcpact-generated equivalent.
- Stable hook-decision receipt shape.
- Adversarial hook corpus for unsafe tool calls and missing approval paths.
- Example showing allow, deny, degraded, and audit-only hook outcomes.

## Steps

1. Decide whether the canonical MCP surface is native or generated by
   `mcpact`.
2. Define the tool list, authority class, and trust ceiling for each MCP tool.
3. Emit one receipt format for hook decisions across all five hook types.
4. Add adversarial fixtures for prompt injection, unsafe path, unsafe network,
   missing approval, and missing doctrine source.
5. Add a smoke command that starts the MCP surface and invokes one read-only
   tool.
6. Add docs showing how a consumer verifies a hook decision receipt.
7. Add catalog evidence only after the MCP smoke and receipt fixtures pass.

## Acceptance evidence

- MCP server starts from the chosen install path.
- At least one allow and one deny receipt are committed as fixtures.
- Adversarial hook corpus has deterministic expected outcomes.
- Docs show the lifecycle from hook event to receipt.
